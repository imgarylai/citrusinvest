## How it's built

<div class="lp-grid not-content">
  <div class="lp-cell">
    <h3>A pure function</h3>
    <p>The core is I/O-free: <code>(strategy spec, data panels) → Report</code>.
    No database, no network, no hidden state — which is what makes it trivially
    embeddable, and easy to test.</p>
  </div>
  <div class="lp-cell">
    <h3>One core, two targets</h3>
    <p>The same crate compiles native (batch backtests parallelized with Rayon)
    and to WASM. This page and the <a href="https://crates.io/crates/yuzu-core">published crates</a>
    run the identical engine — there is no JavaScript reimplementation to drift.</p>
  </div>
  <div class="lp-cell">
    <h3>A DSL, not a framework</h3>
    <p>lemon parses to a JSON expression tree; the engine evaluates it over
    price/fundamental panels. Even this page's <em>syntax highlighting</em> comes
    from the engine's own lexer, compiled to WASM.</p>
  </div>
  <div class="lp-cell">
    <h3>Bring your own data</h3>
    <p>The engine ships no market data and never phones home. Feed it panels
    from any source you're licensed to use — see the
    <a href="guides/bring-your-own-data">data guide</a>.</p>
  </div>
</div>

## A strategy is a file

A complete strategy — *“hold the 30 cheapest names in the S&P 500,
point-in-time”* — is a single text file, not a class to subclass or a notebook
whose result depends on cell-execution order:

```text
#! universe: 20180101..20241231
#! index: sp500
#! config: { "fee_ratio": 0.001 }
#! data-source: fmp
is_largest(rank(-pe), 30)
```

<ul class="lp-file not-content">
  <li><strong>Diffable &amp; reviewable.</strong> It's text — commit it, open a
  PR on it, paste it into an issue. A strategy change is a line change.</li>
  <li><strong>Reproducible bit-for-bit.</strong> The file carries its own
  window, universe, and fees, so a colleague running it on the same data gets
  the same <code>Report</code> — native or WASM.</li>
  <li><strong>No surprise I/O.</strong> A <code>.lemon</code> file can't touch
  the network on its own; data is fetched only when <em>you</em> pass
  <code>--sync</code> with <em>your</em> key. You share a strategy, never a
  credential or a side effect.</li>
</ul>

<p class="lp-file-foot not-content">The alternatives make a strategy
<em>code</em>: backtrader a ~200-line Python class, vectorbt a notebook with
hidden state, Lean a whole C# project. lemon makes it <em>data</em> — which is
why the same file runs in your terminal, in CI, and in the browser above.{' '}
<a href="/guides/playground-to-real-data">Run one on your data →</a></p>

## Get it

**Use the tool, or embed the engine.** The `lemon` CLI runs a `.lemon` file
end-to-end from your terminal; the language bindings drop the *same* engine into
your Rust, Python, or JS/WASM code.

<Tabs>
  <TabItem label="CLI">
    Install the `lemon` binary with one line — macOS / Linux (on Windows, grab
    the [release asset](https://github.com/citrusquant/citrusquant/releases)):

    ```bash
    curl -fsSL https://citrusquant.com/install.sh | sh
    ```

    A strategy is a single file. Save this as `momentum.lemon`:

    ```text
    #! universe: 20180101..20241231
    #! symbols: AAPL, MSFT, NVDA, AMZN, GOOGL, META, JPM, XOM
    #! config: { "fee_ratio": 0.001 }
    #! data-source: fmp
    is_largest(pct_change(close, 63), 3)
    ```

    Run it. `--sync` fetches the declared names' daily bars from the vendor the
    file names — bring your own `$FMP_API_KEY`; nothing is fetched without it:

    ```bash
    export FMP_API_KEY=…          # your key; the file declares the vendor
    lemon momentum.lemon --sync
    ```

    It prints the full `Report` as JSON — the same metrics, equity curve, and
    trades the playground draws, from the same engine. The front-matter keys
    (`universe`, `symbols`, `index`, `config`, `data-source`, …) are in the
    [lemon reference](/reference/lemon).
  </TabItem>
  <TabItem label="Rust">
    ```bash
    cargo add yuzu-core lemon-lang
    ```

    ```rust
    let spec = lemon::parse("is_largest(sma(close, 2), 3)")?;
    let report = yuzu_core::run_backtest(&spec, &ctx, "close", &cfg)?;
    ```

    Or clone and run the self-contained example — the best 40 lines to read first:

    ```bash
    git clone https://github.com/citrusquant/citrusquant
    cargo run -p yuzu-core --example basic_backtest
    ```
  </TabItem>
  <TabItem label="Python">
    ```bash
    pip install yuzu-backtest
    ```

    ```python
    import yuzu

    report = yuzu.run_backtest(
        "close > sma(close, 20)",   # lemon source, straight in
        panels={"close": df_close}, # pandas/polars DataFrame (dates × symbols)
        config={"fee_ratio": 0.001},
    )
    report["metrics"]["sharpe"]
    ```

    Same Rust core underneath (abi3 wheels, Python ≥ 3.9) —
    see <a href="https://github.com/citrusquant/citrusquant/tree/main/crates/yuzu-py">crates/yuzu-py</a>.
  </TabItem>
  <TabItem label="Browser / Workers">
    ```bash
    npm install @citrusquant/yuzu-wasm @citrusquant/lemon-wasm
    ```

    The wasm-pack artifacts this very page runs on
    ([`@citrusquant/yuzu-wasm`](https://www.npmjs.com/package/@citrusquant/yuzu-wasm),
    [`@citrusquant/lemon-wasm`](https://www.npmjs.com/package/@citrusquant/lemon-wasm)).
    They work in browsers and Cloudflare Workers — see the <a href="/reference/api">API reference</a>.
  </TabItem>
</Tabs>

## What it is — and isn't

<div class="lp-honest not-content">
  <div class="lp-is">
    <h3>It is</h3>
    <ul>
      <li>A daily-bar, portfolio-level backtest engine for cross-sectional and trend strategies</li>
      <li>A small, readable strategy language with a complete <a href="reference/lemon">one-page spec</a></li>
      <li>Deterministic and I/O-free — same input, same <code>Report</code>, native or WASM</li>
      <li>MIT-licensed: <a href="https://crates.io/crates/yuzu-core">crates</a>, <a href="https://pypi.org/project/yuzu-backtest/">PyPI wheels</a>, npm WASM packages</li>
    </ul>
  </div>
  <div class="lp-isnt">
    <h3>It isn't</h3>
    <ul>
      <li>Not a data vendor — you bring panels you're licensed to use</li>
      <li>Not tick or order-book simulation — bars in, portfolio NAV out</li>
      <li>Not a broker or execution layer — it stops at the report</li>
      <li>Not investment advice — it computes; you decide</li>
    </ul>
  </div>
</div>
